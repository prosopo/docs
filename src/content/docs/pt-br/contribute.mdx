Recebemos contribuições de qualquer tamanho e colaboradores de qualquer nível de habilidade. Como um projeto de código aberto, acreditamos em retribuir aos nossos colaboradores. Teremos prazer em ajudar com orientação sobre pull requests, redação técnica e transformar qualquer ideia de recurso em realidade.

Quer se envolver ainda mais? Veja nosso [documento de Governança](https://github.com/prosopo/.github/blob/main/GOVERNANCE.md) para descrições detalhadas de diferentes funções, processos de nomeação de mantenedores, processos de revisão de código e aplicação do Código de Conduta.

## Formas de Contribuir

### Repositórios do projeto

Existem muitas formas de contribuir para o Projeto Prosopo! Cada repositório Prosopo tem um README com um link para um arquivo `CONTRIBUTING.md` na raiz do projeto.

Visite o [perfil da Prosopo no GitHub](https://github.com/prosopo) para encontrar os repositórios para:

- O [código principal do Prosopo Procaptcha](https://github.com/prosopo/captcha).

- [Prosopo Docs](https://github.com/prosopo/docs), um site completo da Prosopo! Contribua não apenas com conteúdo escrito, mas também com código Prosopo abordando questões de a11y, CSS, UI e UX. Também disponibilizamos nossa documentação em vários idiomas, então precisamos de ajuda para traduzir todo o site.

### Tipos de contribuições

Além de contribuir com seu próprio código ou conteúdo, você também pode fazer uma grande contribuição se envolvendo ao deixar comentários de revisão em pull requests, adicionando ideias em Issues e Discussões existentes no GitHub, e participando de nossas tarefas de manutenção de issues "Fixadas"!

Todo pull request, especialmente pull requests de tradução, precisa de revisores! Revisar pull requests e deixar comentários, sugestões ou um "LGTM!" ("Looks Good To Me!") de aprovação é uma ótima maneira de começar em qualquer repositório e aprender mais sobre a Prosopo.

Também temos uma comunidade ativa no [Discord](https://discord.gg/3nMYAHecZT)! Valorizamos as contribuições daqueles que dão as boas-vindas a novos membros e respondem a perguntas de suporte.

## Contribuindo para a Documentação

Por favor, envie um pull request em [Prosopo Docs](https://github.com/prosopo/docs).
