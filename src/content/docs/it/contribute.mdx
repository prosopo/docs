Accogliamo contributi di qualsiasi dimensione e contributori di qualsiasi livello di competenza. Come progetto open-source, crediamo nel restituire ai nostri contributori. Siamo felici di aiutare con la guida su PR, scrittura tecnica e trasformare qualsiasi idea di funzionalità in realtà.

Vuole essere ancora più coinvolto? Consulti il nostro [documento di Governance](https://github.com/prosopo/.github/blob/main/GOVERNANCE.md) per descrizioni dettagliate dei diversi ruoli, processi di nomina dei maintainer, processi di revisione del codice e applicazione del Codice di Condotta.

## Modi per contribuire

### Repository del progetto

Ci sono molti modi per contribuire al Progetto Prosopo! Ogni repository Prosopo ha un README con un link a un file `CONTRIBUTING.md` nella radice del progetto.

Visiti [il profilo GitHub di Prosopo](https://github.com/prosopo) per trovare i repository per:

- Il [codebase principale di Prosopo Procaptcha](https://github.com/prosopo/captcha).

- [Prosopo Docs](https://github.com/prosopo/docs), un intero sito web Prosopo! Contribuisca non solo con contenuti scritti, ma anche con codice Prosopo che affronta questioni di a11y, CSS, UI e UX. Rendiamo anche la nostra documentazione disponibile in diverse lingue, quindi abbiamo bisogno di aiuto per tradurre l'intero sito.

### Tipi di contributi

Oltre a contribuire con il suo codice o contenuto, può anche dare un enorme contributo coinvolgendosi lasciando commenti di revisione su PR, aggiungendo idee in Issues e Discussions GitHub esistenti, e partecipando alle nostre attività di manutenzione issue "Pinned"!

Ogni PR, specialmente le PR di traduzione, ha bisogno di revisori! Rivedere PR e lasciare commenti, suggerimenti o un "LGTM!" ("Looks Good To Me!") di approvazione è un ottimo modo per iniziare in qualsiasi repository e per saperne di più su Prosopo.

Abbiamo anche una comunità [Discord](https://discord.gg/3nMYAHecZT) attiva! Apprezziamo i contributi di coloro che accolgono nuovi membri e rispondono alle domande di supporto.

## Contribuire alla documentazione

Si prega di inviare una richiesta PR in [Prosopo Docs](https://github.com/prosopo/docs).
