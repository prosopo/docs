Damos la bienvenida a contribuciones de cualquier tamaño y contribuyentes de cualquier nivel de habilidad. Como proyecto de código abierto, creemos en retribuir a nuestros contribuyentes. Estamos encantados de ayudar con orientación en PRs, redacción técnica y convertir cualquier idea de característica en realidad.

¿Quiere involucrarse aún más? Consulte nuestro [documento de gobernanza](https://github.com/prosopo/.github/blob/main/GOVERNANCE.md) para descripciones detalladas de diferentes roles, procesos de nominación de mantenedores, procesos de revisión de código y aplicación del Código de Conducta.

## Formas de contribuir

### Repositorios del proyecto

¡Hay muchas formas de contribuir al Proyecto Prosopo! Cada repositorio de Prosopo tiene un README con un enlace a un archivo `CONTRIBUTING.md` en la raíz del proyecto.

Visite el [perfil de GitHub de Prosopo](https://github.com/prosopo) para encontrar los repositorios para:

- El [código base principal de Prosopo Procaptcha](https://github.com/prosopo/captcha).

- [Prosopo Docs](https://github.com/prosopo/docs), ¡un sitio web completo de Prosopo! Contribuya no solo con contenido escrito, sino también con código de Prosopo que aborde preocupaciones de a11y, CSS, UI y UX. También hacemos que nuestra documentación esté disponible en varios idiomas, por lo que necesitamos ayuda para traducir todo el sitio.

### Tipos de contribuciones

Además de contribuir con su propio código o contenido, también puede hacer una gran contribución involucrándose dejando comentarios de revisión en PRs, agregando ideas en Issues y Discusiones de GitHub existentes, y participando en nuestras tareas de mantenimiento de issues "Fijados".

Cada PR, especialmente los PRs de traducción, necesita revisores. Revisar PRs y dejar comentarios, sugerencias o un "LGTM!" aprobatorio ("¡Me parece bien!") es una excelente manera de comenzar en cualquier repositorio y aprender más sobre Prosopo.

También tenemos una comunidad activa de [Discord](https://discord.gg/3nMYAHecZT). Valoramos las contribuciones de quienes dan la bienvenida a nuevos miembros, responden preguntas de soporte.

## Contribuir a la documentación
